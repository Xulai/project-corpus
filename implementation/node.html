Node

Our node client listens to uplinks from The Things Network and saves the recieved data into InfluxDB

If the schema isn't set up on InfluxDB to store the datapoints, then on first run the node client creates the schema.

To prevent unknown devices from possibly sending uplinks, the node client gets the list of setup devices from TTN which it then uses as a whitelist and ignores unknown device IDs. If it is unable to get the list then instead relies on a literal array whitelist.

Once the client receives an uplink from TTN, it does a check on type to save Readings, Still Heres and Errors into different measurements on InfluxDB.

If the data uplink is of type readingit uses redis to do some basic processing utilising the previous reading to calculate differences.

To help testing, there's also a data seeder to generate fake data for the InfluxDB measurements. And there are also jasmine tests for the payload formating functions.
