Node

Our node client listens to uplinks from The Things Network and saves the recieved data into InfluxDB.

If the schema to store the datapoints isn't set up on InfluxDB, then on first run the Node client creates the schema.

To prevent unknown devices from possibly sending uplinks, the node client gets the list of setup devices from TTN which it then uses as a whitelist and ignores unknown device IDs. If it is unable to get the list then instead relies on a literal array whitelist.

Once the client receives an uplink from TTN, it does a check on its type to save either in the 'readings', 'still heres' or 'errors' measurement in InfluxDB.

If the data uplink is of type 'reading', it uses Redis to do some basic processing utilising the previous reading to calculate differences. Redis is an in-memory data structure store used as a database and cache.

For testing purposes, there's also a data seeder to generate fake data for the InfluxDB measurements. Using JavaScript testing framework Jasmine, we also provided tests for the payload formatting functions.
