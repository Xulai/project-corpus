Integration testing

This page will display a range of tests that we carried out throughout our project to ensure that our ultrasonic sensor was working as expected (initially on its own, and eventually with all connected components).

Testing our ultrasonic sensor functionality on the river

Method:

Initially, we tested our ultrasonic sensor against an actual river level to confirm that our river simultation logic was working (moving the sensor up indicated river level decreasing and moving it down indicated that the sensor was closer to the top of the river therefor river level was increasing).

Testing our ultrasonic sensor functionality against moving objects

Method:

We noticed that testing for expected values when moving the sensor closer towards floors/walls indoors produced at times anomalous results, therefore we took the alternative approach of testing it against moving objects (simulation of river level moving closer towards it). By balancing the sensor on a table, we moved a wooden board towards it and observed the values printed to the serial monitor.

This time we entered '0' as the river's depth (in mm), and hoped that the river depth values would increase the closer the board got to it. As we can see from the video, this is exactly what the test achieved.

Testing for expected values using an arm for river simulations
